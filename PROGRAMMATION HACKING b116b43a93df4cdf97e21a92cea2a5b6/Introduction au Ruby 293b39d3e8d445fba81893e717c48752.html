Variables et objet simple

Les variables sont des outils puissants utilisés dans beaucoup de langage. Un variable permet de stocké des informations comme du texte, des valeurs, des nombre.

Exemple: On peut dire à Ruby que la variable A=5. Si l'on dit que la variable B=A+5, alors la valeurs de A sera prise en compte et B sera égal à 10.

Pour appelé/ crée une variable, on la nomme et la suis du symnole = suivie de la valeur/ objet voulue. Exemple:Ici la variable "Variable" contient la valeur numérique 45

variable=45

Ici la variable "Variable2" contient le texte "Bonjour"

variable2= "Bonjour"

On peut intégrer un variable dans une variable:

prix_robe=70
prix_chapeau=15
prix_total= prix_robe + prix_chapeau

Organisation par tableaux/hashes

Les tableaux et les hashes ont des façons de grouper et d'organiser plus facilement certaines données.

  • Les tableaux

L'utilisation des tableaux est simple. On stock le tableau dans une variable. Le tableau est symbolisé par des crochet et se qui se trouve entre sont les donnée de ce tableau. On sépare les données par des virgules. Exemple:

mon_tableau = [1,6,8,4]

Ici la variable contient un tableau qui contient les données 1, 6, 8, et 4. Pour afficher le contenu de ce tableau il suffit d'appelé la variable:

mon_tableau

et la console nous renvoie:

=> [1, 6, 8, 4]

Ce tableau nous permet d'utiliser des méthode pour le modifier comme:

mon_tableau.size

qui va m'informer de la taille du tableau ou encore

mon_tableau.reverse

qui va inverser les valeurs du tableau

On peut afficher une des valeurs du tableau en invoquant la variable de ce dernier suivie de la positions de la valeurs entre croche .En ruby, on compte à partir de 0. Donc pour par exemple choisir la valeur 4 de notre tableau, on doit appelé la 3ème valeur:

mon_tableau[3]

Pour ajouter une nouvelle donnée au tableau il faut taper:

mon_tableau << "nouvelle donnée à rentrer"

Et pour remplacer une donnée il faut taper:

mon_tableau[numéro de la donnée à modifier] = "nouvelle valeur"

  • Les tables de hashage

Les tables de hashage permettent un regroupement plus facile que les tableaux. Voici comment les utiliser:

mon_hash = { jean: 10, patrick: 35}

Ici "mon_hash" est le nom de la variable contenant le hash. Le hash est contenue entre deux accolades et les donnée différente sont séparé par une virgule. On stock pour chaque prénom leurs age. Contrairement au tableau ou, pour afficher une valeur, il faut appelé sont numéro, avec les hash on peut directement afficher avec le nom.

mon_hash[:jean]

Cette commande vas nous donner toutes les information stocké sous le nom de jean.

On peut aussi rajouter une valeur dans la table avec:

mon_hash[:marc] = 30

Les conditions

Les conditions permette d'agir en fonction d'une valeur /information. En français, on peut traduire un code de condition par "Si ceci alors cela". On crée d'abord un fichier ruby appelé conditions.rb

  • Synthaxe de condition

On commence par attribuer une valeur à une variable:

a=15

On dit au code de nous dire la valeur de a:

puts "a est égal à: #{a}"

On peut tester le programme pour vérifier que tous fonctionne.

Maintenant on va poser des condition au programme pour qu'il nous indique si a est supérieur ou inférieur à 10. Pour cela on va utiliser "if" et "else" ( si et sinon ):

a = 15
puts "a est égal à: #{a}"
if a>10
puts "Supérieur à 10"
else
puts "Inférieur à 10"
end

  • Conditions dans les tableaux

Pour commencer on crée un tableau pour l'exemple:

emails = [
"bob@bob.com", "jane@hello.com"
]

A la suite on peux écrire un condition qui nous indique si on a 1 email dans le tableau:


if emails.size == 1
	puts "J'ai un email"
else
	puts "J'ai plus ou moin d'un email"
end

La console en executant le code va vous renvoyer "J'ai plus ou moin d'un email". Si l'on en supprime 1 dans le tableau, la console va nous renvoyer "J'ai un email"

  • Symboles de conditions utile

On a deja vu le "==" qui sert à vérifier une égalité mais il en existe d'autre:

! = qui sert à vérifier une différence (souvent traduit en math par ≠)

< et > qui servent à vérifier un supériotité et une infériorité.

&& qui sert à ajouter un "et" logique ( par exemple: if a>1 && a<10 cette condition vérifie un nombre compris en 1 ET 10 )

| | qui sert à ajouter un "ou" logique ( par exemple: if a>1 | | a<10 cette condition vérifie un nombre soit supérieur à 1, soir inférieur à 10)

  • Booleens

Les booleens sont une manière de comparer rapidement des données. Il renvoie automatiquement true ou false (vrai ou faux en francais) en fonction de leurs contenue. Exemple:

a = 15 
mon_booleen =(a > 10)
puts mon_booleen

Ici, a est supérieur à 10, le booleen vas donc répondre true.

Les booleens est une manière plus simple de vérifier une condition mais moin précise.

La programmation orientée objet

  • Les instances de classes

Une classe se rapproche du fonctionnement d'un plan. Elle décrit en quelque sorte le concept de l'objet. On peut afficher la classe d'un objet en tapant l'objet suivi de ".class "

75.7 .class

L'irb nous renvoie:

=> Float

ce qui signifie que l'objet est un nombre flottant ( à virgule)

Exemple: Crée une classe utilisateur:

Comme d'habitude créez un fichier ruby (ici class.rb) et ouvrez-le. Ouvrez à coté votre console. Pour crée une classe "Utilisateur" dans ruby c'est très simple. Il suffit de rentrer:

class Utilisateur

end

La partie entre "class" et "end" sert à définir la classe. Ce qui s'écrit après sert à l'utiliser.

Nous allons maintenant crée un objet de cette classe et dire au script de l'afficher:

class Utilisateur

end 

mon_utilisateur = Utilisateur.new
puts mon_utilisateur

Ruby nous renvoie le nom de la classe suivie de caractères étranges car cette dernière ne contient rien. Pour ajouter une propriété, il faut rajouter l'attribu "attr_accessor :" suivi du nom de l'option:

class Utilisateur
attr_accessor :prenom
end 

mon_utilisateur = Utilisateur.new
puts mon_utilisateur

Notre classe indique maintenant que chaque objet répondant à elle contient un prénom. Pour ajouter un prénom et un nom à notre utilisateur et l'afficher à l'execution du script, il faut taper:

class Utilisateur
attr_accessor :prenom
attr_accessor :nom
end 

mon_utilisateur = Utilisateur.new
mon_utilisateur.prenom = "Bob"
mon_utilisateur.nom = "Rousseau"
puts mon_utilisateur.prenom
puts mon_utilisateur.nom

La console affichera donc "Bob Rousseau"

  • Les méthodes sans paramètres

La façon la plus intéressante pour utiliser les méthodes sont dans les classe. Reprenons la classe de l'exemple d'avant:

class Utilisateur
	attr_accessor :prenom , :nom
end

bob= Utilisateur.new
bob.prenom = "Bob"
bob.nom = "Lenon"

Ici, pour afficher le nom et le prenom et il faut rentrer:

puts bob.nom
puts bob.prenom

Mais cela prends deux ligne. Dans notre cas ce n'es pas problématique mais si l'on rentre une dizaine de paramètre dans la classe, les afficher peut rapidement devenir long. Un méthode peut regrouper plusieurs paramètre afin que, dans notre, exemple, s'affiche le nom et le prenom de notre exmple en tapant: 

puts bob.nom_complet

La création et la définition d'une méthode se fais dans l'entête de notre classe ( entre "class" et "end"):

class Utilisateur

	attr_accessor :prenom , :nom
	def nom_complet
end

end

bob= Utilisateur.new
bob.prenom = "Bob"
bob.nom = "Lenon"

Il suffit maintenant de la définir et de l'appeler à la fin du programme:

class Utilisateur
	attr_accessor :prenom, :nom

	def nom_complet
		nom + " " + prenom
	end
end

bob = Utilisateur.new
bob.nom = "Lenon"
bob.prenom = "Bob"

puts bob.nom_complet

  • Les méthodes avec paramètre

L'exemple suivant nous montrait comment utiliser une méthode dans une classe. Comme les classe, les méthode peuvent imbriquer des paramètres. Pour l'exemple nous définirons un pays et nous demanderons si un utilisateur habite dans ce pays ou un autre.

Pour cela nous ajoutons le paramètre pays à notre code précédant :

class Utilisateur
	attr_accessor :prenom, :nom, :pays
	def nom_complet
		nom + " " + prenom
	end

end

bob = Utilisateur.new
bob.nom = "Lenon"
bob.prenom = "Bob"
bob.pays = "France"

Nous voulons savoir si Bob habite en France. Pour cela, on crée une méthode :

def habite_en(valeur)
valeur == pays
end

Cette méthode appelle la variable "valeur". La suite du code indique que si la valeur correspond au pays rentré dans le paramètre "pays" de l'utilisateur, alors la console retournera vrai ( ou true en anglais). Voici le code final:

class Utilisateur
attr_accessor :prenom, :nom, :pays
	
	def nom_complet
		nom + " " + prenom
	end
	
	def habite_en(valeur)
		valeur == pays
	end
end

bob = Utilisateur.new
bob.prenom = "Bob"
bob.nom = "Lenon"
bob.pays = "France"

puts "Est ce que Bob habite en Suisse ?"
puts bob.habite_en("France")

Si l'on remplace "France" au deux dernières lignes par "Suisse", alors la comparaison de la méthode ne sera plus vérifier et la console nous retourna "true". La comparaison se fais sous les booleen False et True. On peut lui donner de la forme en utilisant "if" et "else":

class Utilisateur
attr_accessor :prenom, :nom, :pays
	
	def habite_en(valeur)
		valeur == pays
	end
 
end

bob = Utilisateur.new
bob.prenom= "Bob"
bob.nom = "Lenon"
bob.pays = "France"

if bob.habite_en("Suisse")
	puts "J'habite en Suisse"
else
	puts "Je n'habite pas en Suisse"
end

En exécutant ce code, le programme vérifie si Bob est Suisse et renverra ça réponse en fonction de la valeur attribuer au paramètre .pays de bob (ici France).

  • L'héritage de classe

Pour bien comprendre ce système d'héritage, nous allons prendre l'exemple d'une classe nommée "animal" que nous allons crée dans ce code:

class Animal
attr_accessor :nom
def initialize(nom)
@nom = nom
end
def parler
puts "Je suis un animal qui s'appelle #{nom}"
end
end
mon_chien = Animal.new("Bob le chien")
mon_chat = Animal.new("Adeline le chat")
mon_chat.parler
mon_chien.parler

Si l'on execute ce programme, la console dira "Je suis un animal qui s'appelle Adeline le chat" et "Je suis un animal qui s'appelle Bob le chien". Mais imaginons que l'on veuille une phrase différente en fonction de l'animal. Cela peut se faire en fesant une classe chat et la duplicquer pour la renommer chien mais cela peut devenir rapidement redondant. On va plutot utiliser l'héritage des classe.

On commence par crée une classe Animal ou l'on va mettre tous les paramètre commun entre le chien et le chat:

class Animal
  attr_accessor :nom
  
  def initialize(nom)
    @nom = nom
  end
end

Puis, on crée la classe Chat et la classe Chien contenant leurs réponse et héritant de la classe Animal grace au symbole "<" suivi de la classe auquel ils héritent:

class Chat < Animal
  
  def parler
    puts "#{nom}: Miaou"
  end
end

class Chien < Animal
  
  def parler
    puts "#{nom}: Ouaf!"
  end
end

mon_chien = Chien.new("Bob le chien")
mon_chat = Chat.new("Adeline le chat")

mon_chat.parler
mon_chien.parler

Le code devrait nous renvoyer:

Adeline le chat: Miaou
Bob le chien: Ouaf!
